## ✨ 登录页功能

### 1. 角色选择

点击顶部的"学生"或"管理员"按钮切换角色。

### 2. 快速测试

页面底部有 3 个测试账号，点击任意一个即可自动填充并登录。

### 3. 手动登录

1. 选择角色
2. 输入账号和密码
3. 点击"登录"按钮

### 4. 记住登录

勾选"记住我"后，下次打开页面会自动登录。

---

## 🔐 登录验证机制

### 优先级顺序

```
1. 尝试调用 Mock API 登录 (3秒超时)
   ↓ 失败
2. 使用本地模拟数据验证
   ↓ 成功
3. 保存到 localStorage
   ↓
4. 根据角色跳转到对应页面
```

### 本地验证逻辑

- 检查账号是否存在
- 检查密码是否正确
- 检查角色是否匹配（学生账号不能登录管理员页面）

---

## 🚪 退出登录

在学生页面或管理员页面：

1. 点击右上角"退出登录"按钮
2. 清除登录状态
3. 返回登录页

---

## 🛡️ 权限控制

### 学生页面

- 检查是否登录
- 未登录 → 跳转到登录页
- 显示当前登录用户信息

### 管理员页面

- 检查是否登录
- 未登录 → 跳转到登录页
- 检查是否是管理员角色
- 非管理员 → 跳转到学生页面

## 🐛 故障排查

| 问题 | 解决方案 |
|------|---------|
| 点击登录没反应 | 检查浏览器控制台 (F12) 是否有错误 |
| 登录后没跳转 | 检查浏览器是否阻止弹窗/重定向 |
| 无法退出登录 | 清除浏览器缓存和 localStorage |
| 提示无权访问 | 确认选择的角色与账号匹配 |

## 📝 技术实现

### 登录状态存储

```javascript
// 保存登录信息
localStorage.setItem('currentUser', JSON.stringify({
  username: 'stu1',
  name: '张三',
  role: 'student'
}));

// 读取登录信息
const user = JSON.parse(localStorage.getItem('currentUser'));
```

### 页面跳转逻辑

```javascript
// 根据角色跳转
if (user.role === 'admin') {
  window.location.href = 'admin-stats.html';
} else {
  window.location.href = 'student-order.html';
}
```

### 权限检查

```javascript
// 页面加载时检查登录
const savedUser = localStorage.getItem('currentUser');
if (!savedUser) {
  window.location.href = 'login.html'; // 未登录，跳转登录页
}
```
